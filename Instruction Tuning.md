# Instruction Tuning

Instruction tuning is the process of [[Fine-Tuning and Alignment|fine-tuning]] a pre-trained [[Large Language Models|language model]] on a dataset of (instruction, response) pairs so it learns to follow human commands rather than merely completing text. It is the transformation that turns a text predictor into an assistant.

## The Problem

A raw pre-trained model predicts the next token. Given a question, it might:

```
Prompt:  "What is the capital of France?"

Raw model: "What is the capital of Germany? What is the capital of Spain?
            What is the capital of Italy?"
            (continues in the same pattern — it's completing a quiz, not answering)

Instruction-tuned: "The capital of France is Paris."
            (understands the intent is to answer, not to continue a list)
```

Pre-training teaches *knowledge*. Instruction tuning teaches *behavior*.

## How It Works

### Training Data Format

```json
[
  {
    "instruction": "Summarize the following article in 3 sentences.",
    "input": "[article text]",
    "output": "The article discusses... Key findings include... The authors conclude..."
  },
  {
    "instruction": "Write a Python function that reverses a string.",
    "input": "",
    "output": "def reverse_string(s: str) -> str:\n    return s[::-1]"
  },
  {
    "instruction": "Translate to French: 'The weather is nice today.'",
    "input": "",
    "output": "Le temps est beau aujourd'hui."
  }
]
```

### Scale

| Model | Instruction Data Size | Method |
|---|---|---|
| InstructGPT (OpenAI, 2022) | ~13K demonstrations | Human-written |
| Alpaca (Stanford, 2023) | 52K examples | Generated by GPT-3.5 ([[Synthetic Data]]) |
| Flan-T5 (Google, 2022) | 1.8K tasks, millions of examples | Aggregated from public NLP datasets |
| Open-source models (2024+) | 100K–1M+ | Mix of human + synthetic |

## Landmark Work

### InstructGPT (2022)

OpenAI's paper that proved the value of instruction tuning:
1. Fine-tune GPT-3 on human-written demonstrations (SFT)
2. Then apply [[Reinforcement Learning from Human Feedback|RLHF]] on top
3. Result: a 1.3B InstructGPT was preferred by humans over the 175B raw GPT-3

The smaller, instruction-tuned model beat the larger raw model — alignment is not just about safety, it's about usability.

### Flan (2022)

Google's "Flan" instruction tuning scaled across 1,800+ NLP tasks:
- Take a pre-trained model (T5, PaLM)
- Fine-tune on thousands of tasks phrased as instructions
- The resulting model generalizes to *unseen* tasks dramatically better

### Self-Instruct (2023)

Wang et al. showed models can generate their own instruction-tuning data:
1. Start with a small seed set of instructions
2. Have the model generate new instructions
3. Have the model generate responses
4. Filter for quality
5. Fine-tune on the generated data

This bootstrapping approach led to Alpaca (Stanford) and catalyzed the [[Open Source AI Models]] instruction-tuning wave.

## The Instruction Tuning Pipeline

```
Pre-trained base model
        ↓
Stage 1: SFT (Supervised Fine-Tuning on instructions)
        ↓ (model can now follow instructions)
Stage 2: RLHF / DPO (preference alignment)
        ↓ (model produces preferred responses)
Stage 3: Safety training
        ↓ (model refuses harmful requests)
Result: Aligned assistant (Claude, ChatGPT, etc.)
```

Instruction tuning is Stage 1 — it's necessary but not sufficient. Without [[Reinforcement Learning from Human Feedback|RLHF]]/DPO, the model follows instructions but may produce harmful, verbose, or low-quality responses.

## What Instruction Tuning Teaches

| Before (raw model) | After (instruction-tuned) |
|---|---|
| Continues text patterns | Answers questions |
| Ignores the user's intent | Follows instructions |
| May generate harmful content | Has basic refusal behavior |
| Outputs in random formats | Formats output as requested |
| No conversation awareness | Understands multi-turn dialogue |

## Instruction Tuning for Code

Code models are instruction-tuned on coding tasks:

```json
{
  "instruction": "Fix the bug in this function",
  "input": "def add(a, b):\n    return a - b",
  "output": "The bug is that the function subtracts instead of adding.\n\ndef add(a, b):\n    return a + b"
}
```

Code instruction tuning data sources:
- Human-written demonstrations
- [[Synthetic Data]] from stronger models
- Code review conversations (PR comments → fixes)
- Stack Overflow Q&A pairs
- Competitive programming solutions with explanations
- [[AI Benchmarks|SWE-bench]] solutions

## Instruction Tuning vs Prompting

| Aspect | Instruction Tuning | [[Zero-Shot and Few-Shot Learning\|Prompting]] |
|---|---|---|
| Changes weights | Yes | No |
| Permanent | Yes | Per-session |
| Cost | Training compute | Per-token |
| Flexibility | Fixed after training | Change prompt any time |
| Quality ceiling | Can exceed prompting | Limited by model's existing capability |
| When to use | Need consistent behavior across all users | Need per-user or per-task customization |

In practice, both are used: instruction tuning creates the base behavior, and [[Prompt Engineering]] / [[Context Engineering]] customizes it for specific use cases.
